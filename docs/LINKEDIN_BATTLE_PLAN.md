# LinkedIn 10K Battle Plan

Target cadence: 7-8 connections/day, personalized, manual-first. Meeting target: 1/week.

## Segment 1 — Financial Decision Makers (II-III category)

- **Who:** CFO, Finance Director, CEO — 1M-70M₾ revenue companies
- **Why:** Bivision sweet spot ✅ FACT (CLAUDE.md). BiFinance = #1 product. CFO pain = manual reporting, Excel dependency.
- **Conversion logic:** connect → "Excel vs BI" message (pain-first) → 15-min call → demo
- **Strategy:** problem-first, not product push. Hook: "4 საათი ხელით რეპორტზე"
- **Volume:** 3/day

## Segment 2 — ERP Partner Ecosystem (hypothesis-first)

- **Who:** IT Directors, COOs, Operations Managers at Balance/Fina/Oris user companies
- **Why (hypothesis):** ERP+BI = natural fit; Balance-style referral credibility transfers across ERP brands. ⚠️ INFERENCE (H) — Balance may be outlier.
- **Conversion logic:** connect → "Balance-Bivision partnership" reference → "თქვენც ხომ Balance-ს იყენებთ?" → integration demo
- **Strategy:** referral-style, partnership credibility. არა cold.

### Validation test (before scaling)

**Test batch:** 20 connections over 2 weeks
- 10 Balance users (control — closest to proven signal)
- 5 Fina users (first generalization test)
- 5 Oris users (second generalization test)

**Success criteria (day 14):**

| Metric | Pass | Fail |
|---|---|---|
| Accept rate | ≥ 40% | < 25% |
| First message reply rate | ≥ 20% | < 10% |
| Call booked | ≥ 2 from 20 | 0 |

**Branch logic:**
- All 3 pass → scale to 2/day, 3-month run
- Balance passes, Fina/Oris fail → narrow to Balance-only, don't generalize
- All 3 fail → ERP ecosystem hypothesis killed, redirect to Seg 1 + Seg 3

**Kill switch:** If after 10 sends accept rate < 15%, stop mid-test. Avoid burning full 20.
