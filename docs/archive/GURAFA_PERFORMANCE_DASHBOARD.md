# Performance Dashboard — როგორ წარუდგინო გუნდს

ვიქტორ, შენი feasibility assessment solid-ია. 3 privacy rule = correct. ახლა coaching lens:

## #1 Rule: არ უთხრა "performance dashboard"

ეს სიტყვა = surveillance. ადამიანი ისმენს "performance" და გულში ფიქრობს "მაკონტროლებენ."

**ნაცვლად:** "გუნდის dashboard" ან "work visibility board" ან უბრალოდ "ჩვენი dashboard."

## როგორ წარადგინო — 3 ნაბიჯი

**ნაბიჯი 1: ჯერ ჰკითხე, მერე აჩვენე.**

კითხვარში (Q5, Q6) ვეკითხებით "რა გჭირდება, რა tool გიშლის ხელს." თუ პასუხებში ამოვიდა "არ ვიცი ვინ რაზე მუშაობს" ან "ინფო ვერ მივიღე" — **მაშინ dashboard = მათი მოთხოვნის პასუხი.** არა ზემოდან დაწესებული, არამედ მათი pain point-ის solution.

**ნაბიჯი 2: გუნდმაც ნახოს.**

ვიქტორმა სწორად დაწერა: "Dashboard ყველასთვის ხილვადი — შენც ხედავ, ისინიც." ეს critical-ია. თუ მხოლოდ ბოსი ხედავს = surveillance. თუ ყველა ხედავს = transparency.

**ნაბიჯი 3: დაიწყე საკუთარი თავით.**

გელას data პირველი ჩანდეს dashboard-ზე. CEO-ს sessions, hours, activity. **ბოსი რომ საკუთარ data-ს აჩვენებს — გუნდი ენდობა რომ fair-ია.**

## რა არ უნდა იყოს

- ❌ Productivity score — subjective, demotivating
- ❌ Screenshots — surveillance
- ❌ Idle detection — ადამიანი არ არის machine
- ❌ Ranking — ინგას და ლუკას შედარება = არაფერს ამბობს

## რა უნდა იყოს

- ✅ "ვინ რაზე მუშაობს ახლა" — visibility, არა control
- ✅ "სად არის bottleneck" — process, არა person
- ✅ "რა გაკეთდა ამ კვირაში" — celebration, არა audit

## ინგას problem

ვიქტორმა სწორად flagged: "ინგა (18 account, ყველაზე loaded) = ყველაზე ნაკლებ ხილვადი."

ინგა Claude Code-ს არ იყენებს, Git-ში არ არის, Qlik-ში account management-ია. **Dashboard-ით ინგას contribution ხილვადი გახდება — ეს მისთვის positive-ია,** არა threatening. "ნახე ინგა 18 account-ს მართავს" > "ინგას idle time 30%."
