## 2. AI-Holic Prevention — ბივიჟენის წესები

### პრობლემა (Ruben Hassid-ის insight)

AI friction-ს ხსნის — ნებისმიერი task "ადვილი" ხდება. შედეგი: მეტი task, მეტი სტრესი, ნაკლები judgment. "ტიპავ prompt-ს და რაღაც არსებობს. არანაირი friction რომ შეჩერდე."

### ბივიჟენის case — ზუსტად ეს მოხდა

| Symptom | ჩვენთან |
|---------|---------|
| Task absorption | 3 დღეში 8+ parallel initiative |
| Work spillage | 12+ საათი sessions |
| No pause to judge | dashboard-ები ≠ decisions |
| More output, less impact | 12 analysis, 0 calls |

### 5 Rule — ბივიჟენისთვის adapted

**Rule 1: "1 sentence before prompt"**
ყოველი სესიის დასაწყისი: "დღეს ვწყვეტ [X] პრობლემას." არა "რა გავაკეთო?" — "რა პრობლემას ვწყვეტ?"

**Rule 2: "Kill 3 per 1"**
AI ახალ task-ს ქმნის? 3 არსებული წაშალე. ჩვენ 10 დღეში 20+ task შევქმენით. 5 საკმარისი იყო.

**Rule 3: "Last prompt = 20:00"**
GP3 burnout rule: 20:00 შემდეგ — არა. System works, შენ ძინავ.

**Rule 4: "Goals, not activity"**
10 docs ≠ progress. 1 call = progress. 1 demo request = progress. Metric = business outcome.

**Rule 5: "Human decides if v1 should exist"**
AI-მ document გააკეთა. კითხვა: "ეს საერთოდ საჭირო იყო?" თუ არა — delete, არა iterate.
